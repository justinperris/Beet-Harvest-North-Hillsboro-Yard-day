GitHub Auto-Sync Token

Paste a Personal Access Token below, then click Save Token. This is what lets Save Now on piler-info update GitHub.

“No expiration” on GitHub is fine. Leave the field empty unless you are entering a new token.

Why status might look empty / Save doesn’t update GitHub

  • Token is only in this browser — not inside the HTML file on disk.
  • Local file vs live site are different.
    Local: file:///…/admin.html
    Live: https://justinperris.github.io/…/admin.html
    You need the token saved on the same place you use piler-info.
  • Chrome and Edge do not share tokens.
  • No expiration is OK. GitHub classic tokens can say “No expiration.”

Create / fix a token (recommended: classic)

  1. GitHub → click your profile photo → Settings
  2. Developer settings (bottom left) → Personal access tokens → Tokens (classic)
  3. Generate new token (classic)
  4. Name: e.g. Piler Tools
  5. Expiration: No expiration is fine (or pick 90 days if you prefer)
  6. Check scope: public_repo
  7. Generate → copy the token once (starts with ghp_)
  8. Paste above → Save Token → Test GitHub Connection
  9. Open piler-info in this same browser and click Save Now

If you use a fine-grained token instead: give it access to repo Beet-Harvest-North-Hillsboro-Yard-day and permission Contents: Read and write (not read-only).
